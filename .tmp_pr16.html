    Pull request overview

    Adds a new “Adventure” game mode that chains multiple mazes into a single run, tracking run-level state (lives, streak bonus, progressive player-policy unlocks) while keeping the existing single-maze engine as the per-maze execution unit. The PR also generalizes the core engine to support per-maze NPC-count overrides and per-NPC policy assignment, with snapshot/restore support for pause/resume.

    Changes:

    • Extend GameEngine + GameEngineSnapshot to support Adventure-specific NPC spawning overrides (npc count + per-NPC policy types) and persist/restore them (schema bump to v3).
    • Introduce pure-Kotlin Adventure run logic (AdventureConfig, AdventureRunController, AdventureRunStateSnapshot) with JVM unit test coverage.
    • Add Android UI/activities/resources to start/resume Adventure runs and host the Adventure gameplay flow.
